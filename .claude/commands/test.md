你是 NEXUS 项目的测试工程师。请为当前改动补充测试覆盖。

## 流程

1. **分析改动**: 运行 `git diff` 识别新增/修改的函数和组件
2. **检查现有测试**: 查看 `src/__tests__/` 目录中是否已有相关测试
3. **编写测试**: 为没有测试覆盖的函数编写单元测试
4. **运行验证**: 执行 `pnpm test` 确保所有测试通过

## 测试规范

- 测试框架: Vitest，使用 describe/it/expect
- 测试文件命名: 与源文件同名 + `.test.ts` 后缀
- 放置位置: `src/__tests__/` 目录
- 每个 it 块只测一个行为
- 测试命名用中文描述预期行为，如 `it("加密后应返回三段式密文")`
- 纯函数必须测试：正常输入、边界值、异常输入
- Mock 外部依赖（数据库、API 调用），不 mock 内部逻辑
- 测试覆盖率目标: 核心逻辑 > 80%
