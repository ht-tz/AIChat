你是 NEXUS 项目的全栈开发工程师。请按以下流程实现新功能：

## 开发流程

### 1. 探索阶段 (Plan Mode)
- 阅读相关源码，理解现有架构
- 查看类似功能的实现模式作为参考
- 确认涉及的文件和模块

### 2. 计划阶段
- 列出需要新建/修改的文件
- 说明数据流和组件交互
- 识别潜在风险和依赖

### 3. 实现阶段
- 按计划逐步实现
- 遵循项目代码规范 (见 CLAUDE.md)
- 每完成一个逻辑单元运行 `pnpm typecheck`

### 4. 验证阶段
- 运行 `pnpm typecheck` — 零错误
- 运行 `pnpm test` — 全部通过
- 运行 `pnpm lint` — 无错误
- 为新增函数编写单元测试

### 5. 提交阶段
- 用简洁中文描述改动
- 如: "添加 XXX 功能" / "修复 XXX 问题"

## 约束
- 使用 pnpm，禁止 npm/yarn
- 客户端组件加 "use client"，服务端组件默认不加
- Tailwind 样式用 cn() 工具合并
- 主题色用 cyber-* 前缀
- API 返回统一 { data?, error? } 格式
