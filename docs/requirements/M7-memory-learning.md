# M7 · Agent 记忆与学习

- **里程碑**：M7

- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

M1-M6 实现了对话、工具调用、推理、多模态、报告、提示词工程。但 Agent 没有"记忆"能力：
- 无法记住跨会话的长期知识
- 无法从历史对话中提取经验
- 无法基于过去经验优化回答

M7 建立记忆与学习系统，让 Agent 能：
- **短期记忆**：当前会话内的上下文理解
- **长期记忆**：跨会话的知识沉淀与检索
- **经验积累**：从成功/失败案例中学习，自动优化

## 2. 用户故事

1. 作为用户，我希望 Agent 记住我之前的对话内容，以便后续对话能基于历史上下文
2. 作为用户，我希望 Agent 记住我的偏好和设定，以便提供个性化服务
3. 作为开发者，我希望查看和管理 Agent 的记忆库，以便调试和优化
4. 作为开发者，我希望 Agent 能从成功案例中学习，以便自动改进回答质量

## 3. 功能范围

### 3.1 包含（In Scope）

- **短期记忆**：会话级别上下文，自动总结和保存当前会话关键信息
- **长期记忆**：跨会话知识存储，支持相似度检索（基于 Mock Embedding）
- **记忆管理**：查看、编辑、删除记忆条目
- **经验积累**：从对话中提取成功经验和失败教训，构建案例库
- **记忆注入**：对话时自动检索相关记忆并注入 prompt
- **记忆可视化**：记忆图谱展示，按主题/时间维度浏览

### 3.2 不包含（Out of Scope）

- 真实 Embedding 模型（默认 Mock，支持切换 OpenAI）
- 记忆压缩算法（M8+）
- 记忆遗忘机制（M8+）
- 多 Agent 共享记忆（M9+）

## 5. 交互 / UI 说明

### 页面结构

- `/memory` - 记忆管理页
  - 左侧：记忆列表（支持搜索和筛选）
  - 右侧：记忆详情（内容、关联会话、重要度、创建时间）
  - 底部：记忆图谱（节点连接展示）

### 关键组件

- `MemoryList` - 记忆列表，支持搜索、筛选、分页
- `MemoryDetail` - 记忆详情面板
- `MemoryGraph` - 记忆图谱可视化
- `ExperiencePanel` - 经验案例面板

### 状态

- 空状态：显示"暂无记忆"引导
- 搜索无结果：显示"未找到匹配记忆"
- 加载中：skeleton 或 loading 动画

## 6. 技术约束

- 数据库：新增 `memories` + `memory_chunks` + `experiences` 表
- Embedding：使用现有 `getProvider().embed()`（Mock 实现）
- 前端路由：Next.js App Router，`/memory` 页面
- 状态管理：Zustand store + localStorage 持久化

## 7. 风险与备选

- **风险**：Embedding 向量维度不一致 → 备选：统一使用 1536 维
- **风险**：记忆检索性能 → 备选：先实现基于向量距离的简单检索
- **风险**：记忆膨胀 → 备选：按重要度排序，只保留高重要度记忆

## 8. 拆分与排期

1. 数据库 schema 扩展（memories + memory_chunks + experiences）
2. 记忆服务层（短期记忆 + 长期记忆 + 检索）
3. 经验积累服务层（经验提取 + 案例库）
4. API 路由（/api/memories + /api/experiences）
5. 前端 Store + 页面 + 组件
6. 导航入口
7. 验证 + 学习文档
