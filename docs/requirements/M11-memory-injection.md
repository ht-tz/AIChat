# M11 · 记忆注入

## 基本信息

- **里程碑**：M11
- **标题**：记忆注入（Memory Injection）
- **负责人**：AI Agent
- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

M7 已经实现了完整的记忆系统（短期记忆、长期记忆、情景记忆、经验积累），但记忆尚未与对话系统集成。用户期望 Agent 能够记住之前的对话内容和知识，在后续对话中自动引用。

M11 实现记忆注入功能：
- **对话记忆注入**：自动检索与当前对话相关的记忆，注入到上下文
- **上下文感知**：根据用户当前问题，智能选择相关记忆
- **记忆更新**：对话结束后，自动将新信息存入记忆系统

## 2. 用户故事

1. 作为用户，我希望 Agent 能记住我之前提到的偏好和信息，以便提供更个性化的服务
2. 作为用户，我希望 Agent 能关联历史对话内容，以便理解上下文相关的问题
3. 作为开发者，我希望记忆注入是可配置的，以便控制注入策略和数量

## 3. 功能范围

### 3.1 包含（In Scope）

- 记忆检索服务：根据当前查询搜索相关记忆
- 记忆注入策略：基于相似度的上下文注入
- 对话结束后记忆更新：自动提取新信息存入记忆
- 记忆注入 API：手动触发记忆检索和注入
- 配置选项：注入数量、相似度阈值、记忆类型过滤

### 3.2 不包含（Out of Scope）

- 记忆压缩算法（后续迭代）
- 记忆遗忘机制（后续迭代）
- 记忆可视化增强（后续迭代）

## 4. 验收标准

- [ ] 记忆注入服务能根据查询检索相关记忆
- [ ] 记忆能自动注入到对话上下文
- [ ] 对话结束后能自动更新记忆
- [ ] API 能手动触发记忆检索和注入
- [ ] `pnpm typecheck` 0 error

## 5. 技术约束

- 使用现有的记忆服务（memory-service.ts）
- 使用现有的向量工具进行相似度计算
- 注入的记忆以 system message 形式添加到上下文
