## 1. 背景与目标

### 项目背景

M14 已完成多智能体协作框架的后端建设，包括工作流引擎、自定义 Agent、消息总线、运行追踪和 SSE 事件流。但目前缺乏直观的前端可视化界面，用户无法实时观察多智能体的协作过程。

### 存在问题

- 多智能体运行过程是黑盒，用户不知道内部发生了什么
- 缺乏可视化的工作流图，难以理解 Agent 间的协作关系
- 步骤输出散落在日志中，没有结构化的时间线展示
- 无法直观感受各 Agent 的工作状态和进度
- 缺少运行控制能力（暂停、继续、停止等）

### 核心目标

将多智能体协作过程以直观的可视化方式呈现，打造沉浸式的协作工作台体验。

### 业务价值

- 提升用户对 AI 协作过程的信任感（可观测）
- 增强产品的科技感和差异化竞争力
- 便于调试和优化工作流配置
- 为后续多人协作、Agent 市场等功能奠定 UI 基础

## 2. 用户故事

- 作为研究人员，我想实时看到多个 Agent 并行工作的状态图，以便了解任务进展
- 作为开发者，我想查看每一步 Agent 的输出详情，以便调试和优化 prompt
- 作为产品经理，我想通过时间线回顾整个协作过程，以便评估工作流效率
- 作为用户，我想随时暂停或停止运行，以便控制成本和进度

## 3. 功能范围

### 3.1 包含（In Scope）

#### 3.1.1 工作流图可视化

- **SVG 节点图**：手工绘制，轻量无依赖
- **节点状态**：待运行 / 运行中 / 已完成 / 失败，四种状态视觉区分
- **连线**：表示 Stage 间的依赖关系
- **布局**：Stage 纵向排列，同 Stage Agent 横向排列
- **动画**：
  - 运行中节点：呼吸发光效果
  - 连线：状态流转时的流光动画
  - 完成节点：勾选动画

#### 3.1.2 实时进度面板

- **总进度条**：基于已完成步骤 / 总步骤
- **阶段指示器**：当前所在 Stage，已完成 Stage 标记
- **运行计时器**：从开始到现在的耗时（mm:ss 格式）
- **状态概览**：运行中 / 已完成 / 失败 / 已取消 的 Agent 数量统计

#### 3.1.3 步骤时间线

- **按时间顺序**展示所有步骤
- 每条时间线卡片包含：Agent 名称、状态、耗时、开始时间
- **可展开/折叠**：点击展开查看完整输出
- **Markdown 渲染**：输出内容支持 Markdown 格式
- **自动滚动**：新步骤出现时自动滚动到底部（可手动暂停自动滚动）

#### 3.1.4 Agent 消息气泡

- **聊天式气泡布局**：左侧是 Agent 头像 + 名称，右侧是消息内容
- **Markdown 渲染**：支持标题、列表、代码块、表格等
- **打字机效果**：新消息逐字显示，模拟实时输出
- **消息类型区分**：
  - 系统消息（灰色）
  - Agent 输出（各角色配色）
  - 错误消息（红色边框）
- **代码块高亮**：支持语法高亮

#### 3.1.5 运行控制

- **开始**：启动新的多智能体运行
- **暂停**：暂停当前运行（后续 Stage 不开始，当前 Agent 继续执行完）
- **继续**：从暂停处恢复
- **停止**：立即终止运行（标记为 cancelled）
- **重跑**：使用相同配置重新运行一次

#### 3.1.6 协作工作台页面

- **页面路径**：`/collaboration`
- **布局**：三栏布局
  - 左侧：工作流图（固定宽度）
  - 中间：消息气泡区（主内容区，可滚动）
  - 右侧：步骤时间线 + 进度面板
- **顶部工具栏**：运行控制按钮 + 团队选择器 + 任务输入框
- **响应式**：
  - 桌面端：三栏布局
  - 平板端：左右可折叠
  - 移动端：Tab 切换

### 3.2 不包含（Out of Scope）

- 可视化工作流编辑器（拖拽配置工作流，下一版本）
- 多人实时协作（同一会话多人观看，后续版本）
- Agent 市场 / 角色商店（后续版本）
- 语音输入输出
- 运行结果导出（PDF/Markdown 导出，后续版本）

## 5. 交互 / UI 说明

### 5.1 设计风格

- **主题**：Cyberpunk / 赛博朋克
- **主色调**：
  - 背景：深色 `#0a0a12`
  - 主色：霓虹青 `#00f0ff`
  - 辅色：霓虹紫 `#b400ff`
  - 强调：霓虹粉 `#ff00a8`
  - 成功：霓虹绿 `#00ff88`
  - 错误：霓虹红 `#ff3366`
  - 文字：`#e0e0e0` / `#888`
- **发光效果**：`text-shadow` / `box-shadow` / `filter: drop-shadow`
- **字体**：等宽字体 + 现代无衬线字体混合

### 5.2 关键组件状态

#### 工作流节点状态

| 状态 | 视觉表现 |
|------|----------|
| 待运行 | 灰色边框，半透明，无发光 |
| 运行中 | 霓虹青边框，呼吸发光动画，内部有旋转加载环 |
| 已完成 | 霓虹绿边框，静态微光，勾选图标 |
| 失败 | 霓虹红边框，错误图标，微闪 |

#### 消息气泡样式

- 每个 Agent 有独立的角色配色
- 头像：圆形，角色图标 + 发光边框
- 消息体：半透明背景，毛玻璃效果，边框发光
- 代码块：深色背景，霓虹语法高亮

### 5.3 关键页面流程

1. **进入页面**：显示空状态，提示选择团队并输入任务
2. **点击开始**：工作流图出现，节点依次点亮，消息区开始输出
3. **运行中**：实时更新状态，用户可随时暂停/停止
4. **完成/失败**：显示最终结果，可重跑或查看历史

## 6. 技术约束

### 依赖的库 / 服务

- `react` — UI 框架（已存在）
- `typescript` — 类型系统（已存在）
- `zustand` — 状态管理（已存在或新增，轻量首选）
- `react-markdown` — Markdown 渲染
- `react-syntax-highlighter` — 代码块语法高亮
- `remark-gfm` — GFM 扩展（表格、任务列表等）

### 影响的模块

- `src/app/collaboration/page.tsx` — 协作工作台页面
- `src/components/collaboration/` — 新组件目录
  - `WorkflowGraph.tsx` — 工作流图
  - `ProgressPanel.tsx` — 进度面板
  - `StepTimeline.tsx` — 步骤时间线
  - `MessageBubble.tsx` — 消息气泡
  - `ControlBar.tsx` — 运行控制栏
- `src/stores/collaborationStore.ts` — Zustand store
- `src/hooks/useMultiAgentSSE.ts` — SSE 连接 hook
- `src/lib/types/multi-agent.ts` — 类型定义（M14 已定义，扩展前端类型）

### 性能 / 安全 / 兼容要求

- SVG 节点数量控制在 50 个以内（合理工作流规模）
- 消息列表虚拟化（超过 100 条时使用虚拟滚动，或渐进式加载）
- SSE 重连机制：指数退避，最多重试 5 次
- 状态持久化：当前运行状态存 localStorage，刷新不丢失
- 无障碍：键盘可操作，语义化 HTML
- 浏览器兼容：Chrome / Edge / Firefox / Safari 最新两个版本

## 7. 风险与备选

| 风险 | 可能性 | 影响 | 应对方案 |
|------|--------|------|----------|
| SVG 动画在低端设备卡顿 | 中 | 中 | 提供性能模式开关，关闭非必要动画 |
| SSE 事件丢失导致 UI 状态不一致 | 中 | 高 | 每次事件带步骤序号，定期用详情 API 校准状态 |
| Markdown 渲染大文本卡顿 | 高 | 中 | 懒渲染 + 代码块折叠，虚拟滚动 |
| 打字机效果实现复杂且性能差 | 中 | 中 | 简化实现：按词而非逐字，用 CSS 动画辅助 |
| Zustand 状态管理过于复杂 | 低 | 中 | 拆分为多个小 store，或改用 React Context + useReducer |

### 备选方案

- **备选 1**：若手写 SVG 太复杂，考虑用 `reactflow` 库（但增加依赖体积）
- **备选 2**：若 Zustand 未在项目中使用，可改用 Context + useReducer
- **备选 3**：若打字机效果性能差，可降级为简单的渐显效果
