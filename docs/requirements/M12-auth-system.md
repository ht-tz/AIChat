# M12 · 权限认证系统

## 基本信息

- **里程碑**：M12
- **标题**：权限认证系统（Authentication & Authorization）
- **负责人**：AI Agent
- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

当前系统对所有用户完全开放，缺少身份认证和访问控制。在生产环境中，必须区分用户身份、保护用户数据、控制资源访问权限。

M12 实现基础权限认证系统：
- **用户认证**：注册、登录、登出、JWT Token 管理
- **API 安全**：认证中间件保护核心 API
- **API 密钥**：为开发者提供程序化访问凭证
- **角色权限**：基础角色（admin / user）权限区分

## 2. 用户故事

1. 作为用户，我希望注册账号并登录，以便保存个人对话和设置
2. 作为开发者，我希望获取 API 密钥，以便通过程序调用接口
3. 作为管理员，我希望管理用户权限，以便控制资源访问

## 3. 功能范围

### 3.1 包含（In Scope）

- 用户注册/登录（邮箱 + 密码）
- JWT 认证中间件
- 密码加密（bcrypt）
- 角色权限控制（admin / user）
- API 密钥生成、列表、删除
- 认证相关 API 路由
- 前端登录/注册页面
- 前端权限控制（路由守卫、按钮显隐）

### 3.2 不包含（Out of Scope）

- OAuth 第三方登录（后续迭代）
- 多租户支持（后续迭代）
- 细粒度 RBAC 资源权限（后续迭代）
- 双因素认证（后续迭代）

## 4. 验收标准

- [ ] 用户可以注册和登录
- [ ] 未认证用户无法访问受保护 API
- [ ] API 密钥可以创建、查看、删除
- [ ] admin 和 user 角色权限不同
- [ ] 密码以加密形式存储
- [ ] `pnpm typecheck` 0 error

## 5. 技术约束

- 使用 bcrypt 进行密码哈希
- 使用 JWT 进行无状态认证
- Token 通过 HttpOnly Cookie 或 Authorization Header 传递
- 默认注册为 user 角色，admin 角色通过环境变量配置初始账号
