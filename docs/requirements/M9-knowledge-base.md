# M9 · 知识库增强

## 基本信息

- **里程碑**：M9
- **标题**：知识库增强（Knowledge Base Enhancement）
- **负责人**：AI Agent
- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

M8 实现了高级推理框架，但推理完全基于实时输入和临时记忆，缺乏：
- 无法接入外部知识库（文档、网页、数据库）
- 无法进行语义检索（RAG）
- 无法多 Agent 协作完成复杂任务

M9 建立知识库增强体系：
- **多数据源接入**：支持文件、URL、数据库查询
- **向量检索**：基于 Embedding 的语义搜索（RAG 核心）
- **多 Agent 协作**：专家 Agent + 协调器模式

## 2. 用户故事

1. 作为用户，我想上传文档并让 Agent 基于文档内容回答问题
2. 作为用户，我想让 Agent 搜索互联网信息并结合知识库回答
3. 作为开发者，我想配置多个专家 Agent 协作解决复杂问题
4. 作为用户，我想查看知识库的文档列表和检索结果

## 3. 功能范围

### 3.1 包含（In Scope）

- 知识库管理：上传文档、查看文档列表、删除文档
- 文档向量化：自动将文档切分为 chunks 并生成 Embedding
- 向量检索：语义搜索、相似文档匹配
- RAG 集成：检索结果注入到 prompt 中
- 多 Agent 协作：专家角色定义、协调器分发任务、结果汇总
- 知识库 UI：文档列表、搜索、检索结果展示

### 3.2 不包含（Out of Scope）

- 实时数据库同步（M10+）
- 知识库权限管理（M10+）
- 知识图谱（M10+）
- 自动知识库更新（M10+）

## 4. 验收标准

- [ ] 能上传文档到知识库并自动向量化
- [ ] 能进行语义搜索并返回相关文档片段
- [ ] RAG 检索结果能注入到对话 prompt
- [ ] 多 Agent 协作能完成复杂任务
- [ ] `/knowledge` 页面能查看和管理知识库
- [ ] `pnpm typecheck` 0 error

## 5. 技术约束

- 使用现有 Mock Embedding（可切换真实模型）
- 文档 chunks 存储到 `memory_chunks` 表（M7 已创建）
- 多 Agent 基于现有 dispatcher 扩展
- 前端路由：Next.js App Router，`/knowledge` 页面
