# M8 · 高级推理

## 基本信息

- **里程碑**：M8
- **标题**：高级推理（Advanced Reasoning）
- **负责人**：AI Agent
- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

M3 实现了基础 ReAct 循环（单步思考→行动→观察），M7 建立了记忆系统。但推理能力仍有限：
- Plan 是扁平列表，无法表达步骤间依赖关系
- 反思只是简单重试，缺乏结构化自省
- 工具选择全靠 Provider 自由发挥，没有策略约束

M8 建立高级推理框架：
- **Plan 优化器**：任务分解 + 依赖图 + 并行执行
- **反思增强**：结构化自省 + 自我修正 + 失败恢复
- **工具选择策略**：基于规则的策略 + 记忆驱动的优化

## 2. 用户故事

1. 作为用户，我希望 Agent 能自动分解复杂任务并识别步骤间的依赖关系
2. 作为用户，我希望 Agent 在出错时能结构化分析原因并自我修正
3. 作为开发者，我想配置工具选择策略（优先/禁止/条件触发）
4. 作为开发者，我想查看推理过程的可视化展示

## 3. 功能范围

### 3.1 包含（In Scope）

- 任务分解（Task Decomposition）：将复杂目标拆分为子任务树
- 依赖图（Dependency Graph）：识别步骤间依赖，支持并行执行无依赖步骤
- 结构化反思（Structured Reflection）：分析失败原因、提出修正方案、自动重试
- 失败恢复（Failure Recovery）：错误分类 + 恢复策略（重试/跳过/降级/中止）
- 工具选择策略（Tool Selection Strategy）：基于规则 + 记忆驱动的工具选择
- 推理可视化（Reasoning Visualization）：展示思考链、Plan 树、工具调用图

### 3.2 不包含（Out of Scope）

- 多 Agent 协作推理（M9+）
- 强化学习训练策略（M10+）
- 人类反馈循环（RLHF）
- 自动 Prompt 优化

## 4. 验收标准

- [ ] Agent 能自动分解复杂任务为子任务树
- [ ] 依赖图能识别并行步骤，无依赖步骤可并行执行
- [ ] 失败时能结构化反思，分析原因并自我修正
- [ ] 工具选择策略可配置（规则 + 记忆驱动）
- [ ] 推理过程可视化展示（思考链 + Plan 树）
- [ ] `pnpm typecheck` 0 error

## 5. 技术约束

- Plan 优化器基于现有 `dispatcher.ts` 扩展
- 依赖图使用邻接表表示，支持拓扑排序
- 工具选择策略存储在 settings store
- 推理可视化复用现有 `MessageBubble` 组件
