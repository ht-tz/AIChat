# M10 · 评估与上线

## 基本信息

- **里程碑**：M10
- **标题**：评估与上线（Evaluation & Launch）
- **负责人**：AI Agent
- **创建日期**：2026-07-03
- **状态**：🚧 进行中

## 1. 背景与目标

M1-M9 构建了完整的 AI Agent 系统，但生产环境还缺少：
- API 安全性（限流、输入验证）
- 文档解析能力（PDF/Word）
- 真实 Embedding 模型接入
- 性能监控和评估体系

M10 完成系统评估和上线准备：
- **安全性加固**：API 限流、输入验证、错误处理
- **文档解析**：PDF 解析、文本提取
- **真实 Embedding**：OpenAI 兼容的 Embedding API
- **性能监控**：响应时间、token 消耗、成功率统计

## 2. 用户故事

1. 作为开发者，我希望 API 有输入验证和限流，以便防止滥用和攻击
2. 作为用户，我希望能上传 PDF 文档到知识库，以便进行文档问答
3. 作为开发者，我希望能使用真实 Embedding 模型，以便提高检索质量
4. 作为开发者，我希望能监控系统性能，以便优化和排查问题

## 3. 功能范围

### 3.1 包含（In Scope）

- API 限流中间件（基于时间窗口的速率限制）
- 输入验证增强（Zod 深度验证、恶意输入检测）
- PDF 文档解析工具（read_pdf）
- 真实 Embedding 接入（OpenAI 兼容 API）
- 性能监控 API（响应时间、token 消耗、成功率）
- 评估指标（回答质量评分、检索准确率）

### 3.2 不包含（Out of Scope）

- 完整的权限系统（后续迭代）
- 多租户支持（后续迭代）
- 实时监控仪表盘（后续迭代）
- A/B 测试框架（已在 M6 基础上扩展）

## 4. 验收标准

- [ ] API 限流正常工作（超过限制返回 429）
- [ ] 恶意输入被正确拦截
- [ ] PDF 文档能正确解析并提取文本
- [ ] 真实 Embedding 模型能正常使用
- [ ] 性能监控 API 能返回统计数据
- [ ] `pnpm typecheck` 0 error

## 5. 技术约束

- 限流使用内存计数器（生产环境可切换 Redis）
- PDF 解析使用开源库（或 Mock 实现）
- Embedding 使用 OpenAI 兼容协议
- 监控数据存储到 localStorage（生产环境可切换 DB）
